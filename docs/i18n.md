# Internationalisation (FR / EN)

Bilingue **français (défaut) / anglais** — différenciateur SEO majeur du marché
Deshaies/Guadeloupe (aucun concurrent direct ne combine schema complet +
bilingue, cf. `seo/strategie.md`).

## Mécanique

- **Routing** : segment `app/[locale]`. URLs explicites `/fr/...` et `/en/...`.
- **`src/proxy.ts`** (ex-middleware Next 16) : une requête sans préfixe est
  redirigée vers la locale détectée via `Accept-Language` (sinon `fr`).
  Exclut `_next`, `api`, `media`, fichiers SEO racine.
- **`generateStaticParams`** (locale + slugs) + `dynamicParams = false` ⇒ tout
  est prégénéré ; locale inconnue ⇒ `notFound()`.
- **Contenu** : `src/content/fr.ts` et `src/content/en.ts` (mêmes clés, type
  `SiteContent`), articles `articles.{fr,en}.ts`. Accès server-only via
  `getContent(locale)` / `getGite` / `getArticle`.
- **SEO** : `pageMetadata()` pose canonical + `alternates.languages`
  (`fr`, `en`, `x-default` → fr). Bascule de langue conservant la page :
  `Header` (desktop) et `MobileBottomNav` (panel).

## Ajouter une page

1. `src/app/[locale]/<route>/page.tsx` :
   - `export const dynamic = "force-static"`
   - `generateMetadata` → `pageMetadata({ locale, path, seo })`
   - corps RSC, `notFound()` si `!isLocale(locale)`
   - `<JsonLd data={breadcrumbLd(...)}/>` (+ schéma spécifique si pertinent)
2. Ajouter les textes dans **`fr.ts` ET `en.ts`** (même structure typée).
3. Ajouter la route à `src/app/sitemap.ts` (`staticPaths`).
4. Si la page est dans la nav : `src/lib/nav.ts` (`mainNav`) et/ou
   `MobileBottomNav` (liste `primary`/`more`).

## Traduire / corriger un texte

Éditer la clé correspondante dans `fr.ts` **et** `en.ts`. Le FR doit rester
**verbatim** du `.com` (cf. `contenu-decisions.md`) ; l'EN est une traduction
professionnelle (pas auto). Ne jamais laisser une clé désynchronisée entre
locales (TypeScript l'impose via `SiteContent`).

## Règles

- Aucune chaîne de contenu en dur dans un composant/page.
- Liens internes via `href(locale, "chemin")` (jamais d'URL absolue codée).
- Un slug d'article/gîte peut différer entre locales (ex.
  `plus-belles-plages-deshaies` / `best-beaches-deshaies`) — c'est voulu pour
  le SEO local de chaque langue ; les `generateStaticParams` les listent par
  locale.
